name: design-foundations
description: Use when choosing color, icons, tokens, or spacing for any UI element — picking a Button/Badge variant, sizing an icon, or deciding padding/gap between elements.
---

Foundation tokens (colour, radius, typography, icon library) are generated by the shadcn preset, not hand-authored. The source of truth is `components.json` + `app/globals.css` — never hand-edit a hex, px radius, or magic spacing value; consume the token.

## Color — pick by role, not hue

1. **Pick by role.** Reach for `brand` or `danger`, never "blue" or "red" — the token decides the color.
2. **One role per surface.** A button is `brand` *or* `danger`, never both.
3. **Neutral is the default.** Reserve colored roles for moments that need them — primary action, warning, error.

| Role | Use for | Button variant |
|---|---|---|
| `neutral` | Default text, secondary UI, body copy | `outline` |
| `brand` | Primary actions, one per view | default |
| `success` | Favourable outcomes — toasts, checkmarks | (no button variant — toasts/badges only) |
| `warning` | Caution to prevent a mistake | `warning` |
| `danger` | Destructive or serious error states | `destructive` |

```tsx
<Button>Create</Button>                       {/* brand */}
<Button variant="outline">Cancel</Button>      {/* neutral */}
<Button variant="warning">Continue anyway</Button>
<Button variant="destructive">Delete</Button>  {/* danger */}
```

## Iconography

1. **Reuse before reaching for new.** One concept maps to one icon everywhere — if "delete" is `Trash` in a table row, it stays `Trash` in the dropdown and the confirmation dialog. Never swap in `XCircle` or `Trash2` for the same concept.
2. **Simplified shapes.** Use the fewest strokes needed to read the metaphor at 16px — dense, multi-line icons (e.g. `TableProperties`) blur at small sizes; prefer the simpler glyph (`Table2`).

Icon library is `lucide-react` (set by the preset). Check the existing icon usage in the codebase before introducing a new glyph for a concept that already has one.

## Spacing — four ranks

Step **one rung up** the ladder per wider container, one rung down per nested level. Use `gap`, not `margin`, for flex/grid children — margin is for prose flow only.

| Rank | Rung | Utility | Use for |
|---|---|---|---|
| Page | 12 (48px) | `space-y-12`, `mt-12` | Between major page sections |
| Card / panel | 6 (24px) | `p-6`, `gap-6` | Inner card padding; rhythm between header/body/footer |
| Fields | 4 (16px) | `space-y-4`, `gap-4` | Between form fields or list rows inside a card |
| Inline | 2 (8px) | `gap-2` | Icon → label inside a button, chip, menu item |

shadcn's `Card` already bakes in rung 6 (`gap-6`, `px-6` on header/content/footer) — don't override it. Your job is rung 4 between fields and rung 2 on inline children.

**Mobile** (`< sm`, 640px): the two outer ranks step down one rung. Fields and Inline don't change — they follow text size, not screen width.

| Rank | Desktop | Mobile |
|---|---|---|
| Page | `space-y-12` | `space-y-8 sm:space-y-12` |
| Card / panel | `p-6` | `p-4 sm:p-6` |
| Fields | `space-y-4` | unchanged |
| Inline | `gap-2` | unchanged |

**Dropdown/menu exception:** items stay at rung 2 (`px-2 py-1.5`, `gap-2`) — never rung 4. Let the hover/selected background separate items, not whitespace.
